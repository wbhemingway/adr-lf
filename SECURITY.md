# Security & Compliance Overview

## 1. Executive Summary
The **ADR Leave Parser** is a specialized desktop utility built to automate the ingestion and parsing of Human Resources (HR) leave forms. Because these documents contain sensitive Personally Identifiable Information (PII) including full names, leave types, and exact dates, this application has been explicitly designed with strict data privacy and isolation principles.

## 2. Core Security Architecture

### 2.1 100% Offline Processing (Air-Gapped)
The primary security feature of the ADR Leave Parser is its completely offline architecture. 
- **No Cloud APIs:** The application does not rely on cloud-based OCR services (e.g., AWS Textract, Google Cloud Vision, Azure AI). 
- **Local Compute:** All Optical Character Recognition is performed entirely on the local CPU using an embedded instance of the open-source Tesseract OCR engine.
- **Air-Gappable:** The application requires zero internet connectivity to function and can be deployed on restricted, offline networks.

### 2.2 Zero Telemetry
The application collects absolutely no analytics, crash reports, or usage telemetry. No data is transmitted externally. 

### 2.3 Local Storage Boundaries
All data artifacts generated by the application are strictly confined to the local filesystem:
- **Application Logs:** Diagnostic logs (`app.log`) do not contain PII and are stored securely in the user's local application data directory (e.g., `%LOCALAPPDATA%\adr-lf` on Windows).
- **Extracted Forms:** The finalized PDF segments and renamed files are saved explicitly to the local output directory designated by the end-user. The application does not cache or retain copies of the PDFs once processing is complete.

## 3. Dependency Security
The software is bundled as a portable executable via Nuitka. The core dependencies include:
- `pytesseract` & `Tesseract-OCR`: Industry standard, open-source OCR engines running purely locally.
- `pdf2image` & `poppler`: Standard libraries for local PDF rasterization.
- `customtkinter`: A pure-Python UI library.

## 4. IT Approval Checklist
- [x] No external network connections required.
- [x] No PII leaves the local machine.
- [x] No background services or daemons installed.
